# 黑卫士 AI 数字档案管理系统真实硬盘接入 SOP

## 原则

真实硬盘接入第一阶段只读接入，不移动、不删除、不改名、不自动改目录。先用小样本跑通，再扩大到完整硬盘。

所有来源最终都要能归集到 NAS：手机、iPad、旧电脑、邮箱、云端、U盘、SD卡、相机、摄像机等先登记、先只读摸底。只有授权确认后，才做完整采集到 NAS 暂存区；暂存区再进入分类、去重、索引和人工复核流程。

图片后续可以进入人脸识别候选池，用来辅助查人、查活动、查项目，但必须先有权限控制。未授权图片只做文件级索引，不做人脸识别。

## 扫描前确认

| 确认项 | 要求 |
|---|---|
| 硬盘状态 | 确认硬盘能正常读取，没有频繁断开 |
| 样本目录 | 先选择一个 5G-50G 的代表性目录 |
| 文件风险 | 有财务、人事、客户证件、合同资料时，先按高密级候选处理 |
| 备份策略 | 先备份索引和看板，不复制大文件 |
| 操作权限 | 由最高授权人确认是否允许打开所在位置、是否允许完整采集到 NAS 暂存区 |
| 图片权限 | 确认是否允许进入人脸识别候选池；未授权只做文件级索引 |

## 真实接入批次

每一次真实接入都必须形成一个批次，避免 40T-60T 数据一次性混在一起。批次用于记录来源、负责人、密级边界、扫描报告和后续是否允许进入 NAS 暂存区。

| 批次字段 | 说明 | 示例 |
|---|---|---|
| 批次编号 | 按来源生成，后续用于追踪和复查 | `USB-2026-001`、`MOB-2026-001`、`NAS-2026-001` |
| 来源类型 | 真实介质或系统来源 | 手机、iPad、旧电脑、邮箱、云盘、NAS、U盘、SD卡、相机 |
| 来源负责人 | 谁提供、谁确认范围、谁负责授权 | 项目经理、部门负责人、最高授权人 |
| 扫描范围 | 本次只读扫描的目录或导出包 | 样本目录、照片目录、邮件包、云盘同步目录 |
| 默认密级 | 未人工复核前的安全等级 | L2 内部候选、L3 敏感候选、L4 机密候选 |
| 扫描前确认 | 设备状态、隐私排除、高密级候选和只读原则 | 硬盘正常、排除个人隐私目录、禁止删除 |
| 扫描报告 | 扫描完成后的结果摘要 | 文件数量、总体积、格式分布、敏感候选、重复候选、读取失败 |
| NAS 入口 | 是否允许进入 NAS 暂存区 | 未授权只索引；授权后进入对应暂存区 |
| AI 候选边界 | 是否允许 OCR、转写、抽帧、人脸识别或案例库候选 | L4-L6 禁训；图片人脸识别必须单独授权 |

批次状态建议分为：`待登记`、`已登记`、`只读扫描中`、`报告待复核`、`允许入 NAS`、`禁止入 AI`、`已归档`。

## 多源接入范围

第一阶段把所有来源先纳入统一索引，不急着把大文件搬到一个地方。系统要记录来源、名称、接入方式、同步策略、同步状态、交叉校验、风险和成本。

| 来源 | 例子 | 第一阶段处理 |
|---|---|---|
| 手机/iPad | iPhone、安卓手机、iPad、旧平板 | 先导出清单或小样本，只读索引；授权后采集到 NAS 暂存区 |
| 旧电脑 | 淘汰 Mac、Windows 旧电脑、旧服务器 | 先按用户和硬盘登记来源，不直接搬空 |
| 邮箱 | 企业邮箱、个人工作邮箱、历史邮件包 | 先索引邮件主题、时间、发件人和附件清单 |
| 在线硬盘 | 当前主机内置盘、长期在线硬盘柜 | 只读扫描，优先形成完整索引 |
| 云端云盘/公有云 | iCloud、OneDrive、百度网盘、阿里云盘、对象存储 | 先接同步目录或清单，不盲目全量下载 |
| 私有云 | Nextcloud、Seafile、企业私有对象存储 | 适合敏感资料远程访问，配合权限和快照 |
| 局域网网盘/NAS | 群晖、QNAP、SMB、AFP、NFS | 建议作为公司私有主库候选 |
| 不同主机硬盘 | 其他 Mac、Windows、Linux、服务器 | 每台主机先本地扫描，再汇总索引 |
| 同主机多接口硬盘 | USB-C、雷电、SATA、DAS、硬盘服务器接口 | 按卷名和接口记录，避免重复扫描 |
| U盘/移动硬盘 | 临时拷贝盘、客户交付盘 | 只做临时索引和风险标记，谨慎入库 |
| SD卡/相机/摄像机 | 相机卡、摄像机卡、运动相机、录音笔卡 | 先保留原始拍摄结构，授权后采集到 NAS 暂存区 |

## 同步与备份取舍

| 策略 | 适用资料 | 当前建议 |
|---|---|---|
| 只读索引 | 历史盘、冷数据、大体积视频 | 第一阶段默认策略，速度最快、成本最低 |
| NAS 暂存区 | 授权采集的手机、旧电脑、邮箱、SD卡、相机、摄像机资料 | 先统一收口，再分类、去重、索引和复核 |
| 云本同步 | 当前项目、轻量文档、常用案例 | 只同步 L0-L2 或已授权资料 |
| NAS 主库 | 公司共享资料、团队协作资产 | 中期建议建设，便于局域网快速访问 |
| 私有云同步 | 需要远程访问的敏感资料 | L2-L4 可选，需要权限和日志 |
| 公有云冷备 | 低密级、重要但低频资料 | 只考虑 L0-L2，先做清单和抽检 |
| 交叉校验 | 合同、经典案例、财务清单、重要成品 | 对路径、大小、修改时间和哈希做对账 |

L4、L5、L6 默认不进公有云，不进入 AI 训练；只允许在本机、NAS 或私有云受控范围内索引和查看。

## 第一步：样本扫描

```bash
npm run scan -- /真实硬盘/样本目录
```

扫描完成后运行：

```bash
npm run verify
```

确认验收报告里的索引文件数、可访问文件数、格式分布和密级分布。
同时检查存储来源、接入方式、同步策略、同步状态、交叉校验、存储风险和成本等级是否识别合理。

## 第二步：页面验收

```bash
npm run dev
```

打开页面后检查：

- “本机索引文件”数量是否等于验收报告数量。
- “本机索引快搜”能否按文件名、路径、格式查到文件。
- 作品类型入口能否筛出方案、标书、合同、图片、视频、录音、表格。
- 点击结果后右侧是否显示路径、大小、修改时间。
- 高级检索能否按存储来源、存储名称、接入方式、同步策略、同步状态、交叉校验、存储风险、成本等级筛选。

## 第三步：补充分类元数据

先按路径和文件名补这几类字段。系统先自动给出候选值，后续允许人工自定义、自我修正。

| 字段 | 逻辑顺序 | 例子 |
|---|---|
| 公司 | 先分公司类型，再写公司名称 | 集团公司、项目公司、临时项目公司；移民公司、广告公司、科技公司、传媒公司、贸易公司，也可以自定义公司名称 |
| 部门 | 先按集团完整建制，再补具体部门 | 董事会、总经办、法务部、财务部、人事部、营销中心、策划部、设计部、项目部、技术部、采购部，也可以自定义部门 |
| 项目 | 先分项目类型，再写项目名称 | 发布会、签证服务、投标项目、广告片、年度报表、年度会议、述职报告、投标文件、客户提案 PPT、内部交流学习 PPT、Word 文档、电子表格、邮件、录音、会议文件、视频文件 |
| 作者 | 先记录创建来源，再记录具体人 | 公司创建者、项目公司、集团公司、部门、具体人员、创建时间；文案、设计、剪辑、顾问、文件创建者 |
| 负责人 | 按责任等级分层 | 项目经办人、项目主理人、项目经理、部门长、分管副总、最高权限人 |
| 周期阶段 | 支持模糊时间段的周期搜索和按周期阶段模糊搜索 | 年、月、日、时；2016 到 2019、2020 上半年、Q1、筹备期、执行期、交付期、归档期 |
| 密级 | 从外部流通到绝密分层 | L0 外部流通、L1 普通、L2 内部、L3 敏感、L4 机密、L5 最高授权、L6 绝密 |

## 第四步：作品状态和价值等级

作品不要只靠一个“评分”判断，需要拆成三套逻辑，后续检索和 AI 喂料才不会混乱。

| 维度 | 逻辑顺序 | 用途 |
|---|---|---|
| 完整状态 | 残缺待补充、残缺已补充、半成品待收尾、完整已归档、待人工确认 | 判断文件是否需要 AI 收尾、人工复核或直接归档 |
| 价值等级 | 常规作品、优质作品、优秀案例、经典案例、示范文档 | 判断是否适合进入案例库、风格库和对外展示 |
| 制度等级 | 惯例制度、经典制度、制度模板、流程规范 | 单独沉淀制度资产，避免和普通作品混在一起 |

推荐顺序：先判断密级，再判断完整状态，再判断价值等级。L4、L5、L6 默认不进入 AI 训练；L0、L1 可以进入外部展示或通用案例；L2、L3 需要授权后再使用。

## 第四步补充：真实介质样本验证

真实硬盘、NAS、手机、iPad、U盘、SD卡、相机、摄像机、云盘和邮箱都不能一上来全量接入。先用一个小样本做验证，确认速度、风险、密级和报告格式都能接受，再扩大扫描。

| 步骤 | 责任人 | 验证关口 | 输出物 |
|---|---|---|---|
| 接入前登记 | 项目负责人 | 确认来源人、设备类型、公司/部门/项目、默认密级和授权范围 | 样本批次编号、来源说明、隐私排除目录、负责人 |
| 只读试扫 | 技术/档案管理员 | 只扫描 1 个小目录或 1 个设备样本，不移动、不删除、不改名 | 文件数量、体积、格式分布、不可读条目、耗时记录 |
| 风险复核 | 部门负责人 | 检查 L4-L6、财务、人事、合同、隐私、密钥、未知来源文件 | 禁训清单、隔离候选、高密级候选、人工复核意见 |
| 批次报告 | 档案管理员 | 确认批次摘要、批次历史、合并索引和浏览器脱敏索引一致 | 批次报告、脱敏页面数据、合并索引摘要 |
| NAS/AI 分流 | 最高授权人/系统管理员 | 授权后才进入 NAS 暂存区；L4-L6 不进公有云、不进 AI 训练 | NAS 暂存位置、AI 候选清单、禁训清单 |
| 扩大接入 | 项目主理人 | 样本验证通过后，按公司、部门、年份或设备分批扩大 | 下一批扫描范围、预计体量、负责人、回滚方案 |

样本验证通过的标准：页面可检索、批次报告可追溯、浏览器数据不含本机完整路径、高密级资料进入禁训边界、负责人能确认下一批范围。

### 真实样本点验登记模板

真实样本扫描前先填登记模板。未填完整，不进入全量扫描。

| 字段 | 填写口径 |
|---|---|
| 样本来源类型 | 硬盘、NAS、手机、云盘、U盘、SD卡、相机、邮箱等 |
| 来源名称 | 设备编号、卷名、云盘账号或邮箱账号 |
| 公司/部门/项目 | 所属公司、部门、项目名称，允许临时项目公司 |
| 负责人 | 项目主理人、经办人、部门负责人、最高授权人 |
| 样本范围 | 先选 1 个小目录、1 个日期段或 1 个设备样本 |
| 模糊时间段 | 如 2018-2020、2024 年会、项目筹备期到收尾期 |
| 隐私排除 | 个人相册、账号密码、密钥、私人聊天、非授权客户资料 |
| 默认密级 | 默认 L2/L3；发现 L4-L6 立即标记禁训和人工复核 |
| 扫描方式 | 只读扫描；不移动、不删除、不改名真实文件 |
| NAS 暂存位置 | 按来源类型、负责人、日期、批次编号进入暂存区 |
| AI 候选边界 | L0-L2 可候选；L3 授权后候选；L4-L6 禁止训练 |
| 验收结果 | 通过、待补充、隔离、暂停、扩大下一批 |

### 真实样本扫描前检查清单

这些关口未通过，不进入真实扫描。

| 检查项 | 检查口径 | 状态要求 |
|---|---|---|
| 只读原则 | 本轮只建立索引和报告，不移动、不删除、不改名真实文件 | 必须通过 |
| 来源授权 | 确认来源人、负责人、公司/部门/项目、授权范围和隐私排除项 | 必须通过 |
| 样本范围 | 先选小目录、日期段或单设备样本，不直接扫描整块 40T-60T | 必须通过 |
| 空间与备份 | 确认本机/NAS 有足够索引空间；扫描前不复制大文件，先备份小索引 | 必须通过 |
| 安全风险 | 未知 U盘、旧电脑、邮件附件先标风险；高风险文件不进入正式库 | 人工复核 |
| 密级禁训 | L4-L6、财务、人事、合同、隐私、密钥默认进入禁训和人工复核 | 人工复核 |
| 路径脱敏 | 浏览器只显示相对路径和编号；本机完整路径只留在本地服务端 | 必须通过 |
| 负责人签字 | 扫描报告、批次摘要、下一批范围必须由负责人确认 | 必须通过 |

## 第五步：扩大扫描

样本目录验收通过后，再按公司或年份分批扫描：

```bash
npm run scan -- /真实硬盘/公司A
npm run scan -- /真实硬盘/2020-2026
```

现阶段每次扫描会生成一份新的当前索引。后续单机可用版需要增加“多目录合并索引”和“SQLite 数据库”，避免大硬盘反复全量扫描。

当前已提供多目录合并索引基础脚本。样本索引稳定后，可以把多个扫描索引合并去重：

```bash
npm run merge -- 界面原型-v1/archive-index.json
```

后续真实接入时，每个来源先保留一份私有索引，再执行合并索引；浏览器只读取脱敏合并结果，真实路径仍留在本机服务端。

## 第六步：授权采集到 NAS 暂存区

只有在样本扫描、页面验收、密级判断都确认后，才进入完整采集。采集不是直接整理正式档案，而是先进入 NAS 暂存区。

| 采集对象 | 采集前确认 | 暂存区处理 |
|---|---|---|
| 手机/iPad | 确认照片、聊天导出、文件目录范围 | 按设备、人员、日期建立批次目录 |
| 旧电脑 | 确认用户、硬盘、系统账号和隐私范围 | 先保留原路径结构，避免遗漏 |
| 邮箱 | 确认邮箱账号、时间范围、附件范围 | 邮件和附件分开索引，保留邮件来源 |
| 云端 | 确认云盘账号和同步范围 | 先采集清单和授权目录，不盲目全量下载 |
| U盘/SD卡 | 确认来源人和用途 | 先复制到暂存区，再做去重和风险标记 |
| 相机/摄像机 | 确认拍摄项目和时间 | 保留原始文件名和拍摄结构，后续再分类 |

进入 NAS 暂存区后，按顺序做：来源登记、完整性检查、分类、去重、索引、密级复核。图片类资料再根据授权进入人脸识别候选池；L4、L5、L6 不进公有云，不进入 AI 训练。

## 第七步：备份索引

每次确认索引可用后运行：

```bash
npm run backup
```

备份快照只复制小文件：索引、看板、验收报告。不会复制真实大文件。

## 第八步：多源对账

真实样本跑通后，再按来源逐批对账：

| 对账对象 | 对账内容 | 建议频率 |
|---|---|---|
| 本机硬盘 | 路径、大小、修改时间 | 每次扫描 |
| 外接硬盘/U盘 | 卷名、路径、大小、抽样打开 | 每次挂载扫描 |
| NAS/局域网网盘 | NAS 清单、本机索引、差异报告 | 每日或每周 |
| 云端云盘 | 云端清单、本机同步目录、修改时间 | 每日或每周 |
| 私有云 | 私有云清单、本机索引、哈希抽检 | 每日或每周 |
| 高价值作品 | 哈希、版本、负责人、密级 | 每次入库或交付前 |

## 风险控制

- 不直接扫描整块 60T 硬盘，先按样本目录验证速度和分类准确度。
- 不把高密级资料喂给 AI，先进入禁训清单。
- 不开放删除权限，删除必须放到后续权限系统。
- 不自动改真实文件名，先在索引里补元数据。
- 不把原始音视频上传外部服务，转写和抽帧优先考虑本机处理。
- 不在未授权时把手机、旧电脑、邮箱、SD卡、相机、摄像机内容完整采集到 NAS 暂存区。
- 不把未授权图片送入人脸识别候选池。
