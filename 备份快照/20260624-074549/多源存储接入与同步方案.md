# 黑卫士 AI 数字档案管理系统多源存储接入与同步方案

## 总原则

不要一开始把 40T-60T 数据全部搬到一个地方。第一阶段只做统一索引、分级接入和交叉校验；真正复制大文件只针对高价值、热数据和必须备份的数据。

最终归集方向：手机、iPad、旧电脑、邮箱、云端、NAS、U盘、SD卡、相机、摄像机等来源都要纳入统一管理。NAS 作为最终归总硬件，但不是一上来直接整理原文件；先建立“NAS 暂存区”，授权后把需要完整采集的资料进入暂存区，再分类、去重、索引，图片再进入人脸识别候选池。

第一阶段不擅自移动、删除、改名真实文件。完整采集到 NAS 暂存区必须经过授权确认，尤其是旧设备、邮箱、相机卡和客户资料盘。

推荐策略：本地硬盘、私有云/NAS、云端云盘三类并行，但不是全量三份复制。热数据同步，冷数据只建索引，敏感数据受控同步。

## 存储来源分类

| 来源 | 例子 | 接入方式 | 初始策略 |
|---|---|---|---|
| 手机和平板 | iPhone、安卓手机、iPad、旧平板 | 数据线、系统备份、照片导出目录 | 先只读清单，授权后采集到 NAS 暂存区 |
| 在线硬盘 | 当前主机内置硬盘、长期在线硬盘柜 | 本机路径只读扫描 | 优先接入，生成完整索引 |
| 外接硬盘 | USB-C 移动硬盘、雷电硬盘柜、多盘位硬盘盒 | 挂载后只读扫描 | 按硬盘分批索引，不自动复制大文件 |
| U盘 | 临时 U 盘、客户拷贝盘 | 挂载后只读扫描 | 只做临时索引和风险标记，谨慎入库 |
| SD卡 | 相机卡、摄像机卡、录音笔存储卡 | 读卡器挂载后只读扫描 | 先建立卡片来源记录，授权后采集到 NAS 暂存区 |
| 相机和摄像机 | 单反、微单、运动相机、摄像机 | 存储卡、数据线或原厂导出工具 | 先保留原始拍摄结构，后续再按项目和日期整理 |
| 邮箱 | 企业邮箱、个人工作邮箱、历史邮件备份 | 邮件客户端导出、EML/MBOX/PST 清单 | 先索引主题、发件人、时间和附件，不擅自公开内容 |
| 局域网网盘 | SMB、AFP、NFS 共享目录 | 网络挂载扫描 | 建立中心索引，定时差异对账 |
| NAS | 群晖、QNAP、TrueNAS、文件服务器 | NAS 共享目录或 API | 建议做私有主库和快照 |
| 不同主机硬盘 | 其他 Mac、Windows、Linux、服务器硬盘 | 远程共享、远程索引汇总 | 各主机本地扫描，再汇总索引 |
| 同主机多接口硬盘 | USB、雷电、SATA、硬盘柜混接 | 按卷名和接口记录 | 防止重复扫描和盘符变化 |
| 云端云盘 | iCloud、OneDrive、Google Drive、Dropbox、百度网盘、阿里云盘 | 同步客户端目录或 API | 不建议全量下载，先同步清单和热数据 |
| 私有云 | Nextcloud、ownCloud、Seafile、企业私有对象存储 | 私有云同步或 API | 高安全资料优先私有云，配合本地快照 |
| 混合云 | 本地 NAS + 私有云 + 公有云冷备 | 分层同步 | 高价值资料多点校验，普通冷数据只留索引 |

## 推荐架构

第零层：多设备归集入口  
手机、iPad、旧电脑、邮箱、云端、U盘、SD卡、相机、摄像机等来源先登记来源和责任人。第一阶段只读摸底；经授权后，完整采集到 NAS 暂存区。

第一层：统一索引层  
所有来源先只读扫描，生成文件名、路径、大小、格式、修改时间、来源、同步状态、密级、作品等级。

第二层：NAS 暂存区  
授权采集后的资料先进入 NAS 暂存区，不直接改成正式档案。暂存区负责承接原始拷贝、来源记录、采集批次和责任人，后续再分类、去重、索引和人工复核。

第三层：热数据同步层  
最近项目、常用案例、待收尾半成品、经典案例进入本机和云端同步。

第四层：冷数据索引层  
历史大文件、低频视频、旧项目素材只保留索引和所在位置，不立刻复制。

第五层：图片和人脸候选层  
图片可以在后续进入人脸识别候选池，用来辅助查找人物、活动和项目，但必须受权限控制。没有授权的图片只做文件级索引，不做人脸识别。

第六层：高密级受控层  
L4、L5、L6 默认不进公有云，不进入 AI 训练，只允许本机、NAS 或私有云受控索引。

## 同步策略

| 策略 | 适用对象 | 优点 | 风险 | 建议 |
|---|---|---|---|---|
| 只读索引 | 冷数据、历史盘、大体积视频 | 快、便宜、不动原文件 | 文件离线时只能查到位置 | 第一阶段默认策略 |
| 本机快照 | 项目看板、索引、小文件清单 | 恢复快、成本低 | 不能保护原始大文件 | 每次扫描后执行 |
| 云本同步 | 当前项目、常用文档、轻量资料 | 多设备可用 | 云端费用和隐私风险 | 只同步 L0-L2 和授权资料 |
| NAS 暂存区 | 授权采集的手机、旧电脑、邮箱、SD卡、相机、摄像机资料 | 统一收口、便于后续整理 | 如果不分批会堆成新乱库 | 先暂存，后分类、去重、索引 |
| NAS 主库 | 公司共享资料、团队协作 | 容量大、局域网快 | 需要设备和维护 | 建议作为私有主库 |
| 私有云同步 | 敏感但需要远程访问的资料 | 权限可控 | 部署成本较高 | L2-L4 可选 |
| 公有云冷备 | 低密级、重要但不常用资料 | 异地灾备 | 成本、合规和隐私风险 | L0-L2 才考虑 |
| 三方交叉校验 | 高价值资料、经典案例、合同清单 | 防丢、防错 | 耗时 | 对清单、哈希、大小、修改时间做校验 |

## 交叉校验规则

| 校验项 | 用途 | 频率 |
|---|---|---|
| 路径清单 | 判断文件是否还在原位置 | 每次扫描 |
| 文件大小 | 快速发现缺失或截断 | 每次扫描 |
| 修改时间 | 判断是否有新版本 | 每次扫描 |
| 文件哈希 | 精确判断重复和篡改 | 重要文件抽检，经典案例全检 |
| 卷名和设备名 | 防止外接盘盘符变化 | 每次外接硬盘扫描 |
| 云端清单对账 | 判断云端和本机是否一致 | 每日或每周 |
| NAS 快照对账 | 判断中心主库是否有变化 | 每日或每周 |

## 成本与安全建议

最省钱方案：本地硬盘只读索引 + 小文件快照 + 重要文件手动备份。

平衡方案：本地硬盘只读索引 + NAS 主库 + 云盘同步热数据。

高安全方案：本地索引 + NAS 快照 + 私有云 + 公有云低密级冷备。

不建议第一阶段做全量三份复制，因为 40T-60T 数据会非常耗时间、耗钱，也会放大隐私和权限风险。

## 接入优先级

1. 当前主机和在线硬盘：最快闭环。
2. 外接硬盘和 U盘：按盘只读扫描，先建索引。
3. NAS 和局域网网盘：建立中心索引和差异对账。
4. 不同主机硬盘：每台主机本地扫描，再汇总索引。
5. 云端云盘：先接清单和同步目录，不盲目全量下载。
6. 私有云和混合云：作为中长期安全架构。

## 第一版落地动作

- 每条索引记录增加存储来源、存储名称、接入方式、同步策略、同步状态、交叉校验、风险等级、成本等级。
- 当前样本先标记为本机硬盘。
- 真实硬盘接入时按来源逐批扫描，不混扫。
- 手机、iPad、旧电脑、邮箱、云端、U盘、SD卡、相机、摄像机先登记来源，授权后再完整采集到 NAS 暂存区。
- NAS 暂存区只做承接和复核，不直接改名、删除或替换真实来源文件。
- 进入暂存区后再做分类、去重、索引；图片后续进入人脸识别候选池，但必须有权限控制。
- 云盘先做清单和同步目录，不做全量下载。
- NAS 优先作为私有主库候选。
- 高密级文件先本地或私有云受控，不进公有云和 AI 训练。
